# Git 配置说明

## 已忽略的敏感文件

以下文件包含敏感信息，不会被上传到 Git：

### 1. 环境配置文件
- `backend/.env` - 包含数据库密码、API Keys 等敏感信息
- `.env.local` - 本地环境变量
- `*.env.*.local` - 其他本地环境配置

**注意**: `backend/.env.example` 会被提交，作为配置模板供其他人参考。

### 2. 虚拟环境
- `.venv/` - Python 虚拟环境
- `backend/.venv/` - 后端虚拟环境
- `venv/`, `env/`, `ENV/` - 其他虚拟环境目录

### 3. 依赖锁定文件
- `poetry.lock` - Poetry 依赖锁定文件（已忽略）
- `backend/poetry.lock` - 后端依赖锁定文件

**说明**: 如果你需要共享确切的依赖版本，可以考虑提交 `poetry.lock`。但在开发阶段通常不需要。

### 4. IDE 配置
- `.idea/` - PyCharm/IntelliJ IDEA 配置
- `.vscode/` - VS Code 配置
- `*.swp`, `*.swo` - Vim 临时文件

### 5. Node.js 相关
- `frontend/node_modules/` - npm 包目录
- `frontend/dist/` - 构建输出
- `*.local` - 本地配置文件

### 6. 日志和临时文件
- `*.log` - 日志文件
- `logs/` - 日志目录
- `backend/logs/` - 后端日志

### 7. 数据库文件
- `*.db`, `*.sqlite`, `*.sqlite3` - SQLite 数据库文件

## 如何添加新的敏感文件

如果你创建了新的包含敏感信息的文件，请确保：

1. 在 `.gitignore` 中添加相应的规则
2. 使用 `git check-ignore -v <文件路径>` 验证文件是否被正确忽略

示例：
```bash
git check-ignore -v backend/.env
```

## 如果不小心提交了敏感文件

如果发现敏感文件已经被提交到 Git，请立即：

1. 从 Git 历史中移除该文件
2. 更改所有泄露的密码和 API Keys
3. 重新提交

命令示例：
```bash
# 从 Git 追踪中移除文件（但保留本地文件）
git rm --cached backend/.env

# 提交更改
git commit -m "Remove sensitive file from tracking"
```

## 推荐的 Git 工作流程

1. **初始化仓库后首次提交**：
   ```bash
   git add .
   git commit -m "Initial commit"
   ```

2. **日常开发**：
   ```bash
   git add .
   git status  # 检查哪些文件会被提交
   git commit -m "描述你的更改"
   ```

3. **查看被忽略的文件**：
   ```bash
   git status --ignored
   ```

## 重要提醒

⚠️ **永远不要提交以下内容**：
- 密码和密钥
- API Keys
- 数据库连接字符串
- 私钥文件（.pem, .key, .crt 等）
- 包含真实数据的配置文件

✅ **应该提交的内容**：
- 源代码
- 配置文件模板（如 .env.example）
- 文档
- 构建脚本
